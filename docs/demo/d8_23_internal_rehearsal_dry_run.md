# D8.23 Internal Rehearsal Dry-Run

- Current state: READY_FOR_STAGING_HANDOFF
- External staging state: WAITING_FOR_REAL_STAGING_EVIDENCE

This dry-run is based on existing D8.18-D8.22 materials and the local PartnerOS demo flow. It is not a real HOSUN, JOOBOO, Chongqing Huiju, or future partner session.

## Timing Assessment

Can the story be completed in 10-15 minutes?

- Internal dry-run result: yes, if the presenter keeps Workbench, Customer Development, and Portal Operations brief.
- Risk: Order Detail, Market Response, and partner-specific talk tracks can consume too much time.
- Recommendation: use one featured order and one sentence each for HOSUN, JOOBOO, and future partner before opening detail pages.

## Workbench Opening

Assessment:

- Workbench can be used as the opening frame.
- It explains daily operation better than starting from a single CRM-like page.
- It should be described as the map for business development, delivery operations, and management decisions.

Compression:

- Keep to 60-90 seconds.
- Do not explain every card.
- Use it to orient the audience, then move to Campaign.

## Growth Operations / Campaign

Assessment:

- Growth Operations / Campaign is clear when framed as a manual campaign workspace.
- It should be explained as segment + product focus + next action + attribution.
- It is important to say that Campaign does not automatically send external messages.

Compression:

- Show one HOSUN example and one JOOBOO example only.
- Avoid editing multiple fields during the partner-facing rehearsal.

## Manual Outreach

Assessment:

- Manual Outreach is understandable if the presenter says: draft generation, follow-up task, manual status only.
- It should not be described as email automation.

Compression:

- Show one Chinese draft and one English draft.
- Show status options only briefly.
- Repeat the no-auto-send boundary once.

## Quote / Order / Production / Shipment

Assessment:

- The Quote -> Order -> Production -> Shipment chain is coherent.
- The strongest page is Order Detail because it shows customer-visible summary, partner split, production, shipment, and feedback in one place.
- Use one featured order instead of browsing multiple orders.

Potential page switching issue:

- Moving from `/orders` to a specific Order Detail can take time if the presenter searches manually.
- Presenter should prepare the featured order route before the session.

## Feedback / Market Response

Assessment:

- Feedback and Market Response can close the loop.
- The presenter should explain that Feedback is internal handling and Market Response is advisory.
- The key message is: repeated feedback and delivery issues inform product and partner focus, but do not automatically change operations.

Compression:

- Open Feedback Tickets briefly.
- Spend more time on Market Response because it explains strategic value.

## HOSUN Talk Track

Assessment:

- HOSUN lifting systems talk track is clear.
- Required terms are ready: lifting systems, desk frames, desk legs, lifting columns, heavy-duty supply.
- Performance dimensions are ready: load, stability, noise, delivery, installation, after-sales, packaging, warranty, test cycle, certification, project demand.

Compression:

- Do not explain all dimensions during the first rehearsal.
- Use three examples: load, noise, delivery.
- Mention the full list as feedback dimensions for follow-up.

## JOOBOO Talk Track

Assessment:

- JOOBOO education furniture talk track is clear.
- Required terms are ready: education furniture, school desks/chairs, project furniture.
- Project dimensions are ready: school procurement timing, delivery consistency, installation, resource needs, feedback after use, project acceptance criteria.

Compression:

- Use school procurement timing and delivery consistency as the first examples.
- Keep product detail short unless JOOBOO asks for deeper workflow mapping.

## Future Partner Talk Track

Assessment:

- Future partner talk track is clear when framed as shared workflow plus partner-specific fields.
- It should mention Chongqing Huiju and future partner onboarding without implying a completed real partner configuration.

Compression:

- Say: any future partner can enter through Partner Onboarding, product family, quote logic, delivery requirement, resource taxonomy, customer-visible fields, and Market Response metrics.

## Pages That Can Slow the Rehearsal

- Order Detail, if the route is not prepared.
- Market Response, if every signal is explained.
- Portal Operations, if staging credentials are discussed too deeply.
- Partner Onboarding, if the presenter explains every readiness field.

2. Campaign and Manual Outreach: 3 minutes.
3. Quote and Order Detail: 3 minutes.
4. Production / Shipment / Portal: 2 minutes.
5. Feedback / Market Response: 3 minutes.
6. Partner Onboarding and close: 2 minutes.
